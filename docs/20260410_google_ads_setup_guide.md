# Google広告 初期設定ガイド

> 対象: CloudNature AI見積もりシミュレーター（ai.cloudnature.jp）
> 前提: Google広告を初めて使う。超ミニマム（月10〜20万円）でGoogle検索広告1本から開始
> ref: `docs/20260409_ad_strategy_master.md`

---

## 全体の流れ

```
Step 1: Google広告アカウント開設
Step 2: コンバージョンアクションの設定（GA4からインポート）
Step 3: キャンペーン作成（検索広告）
Step 4: 広告グループ・キーワード設定
Step 5: 広告文（RSA）の作成
Step 6: 広告表示オプションの設定
Step 7: 除外キーワードの設定
Step 8: 最終確認・配信開始
```

---

## Step 1: Google広告アカウント開設

1. https://ads.google.com/ にアクセス
2. 「今すぐ開始」→ Googleアカウントでログイン
3. **重要: 「エキスパートモードに切り替える」を選択**
   - デフォルトの「スマートモード」は自動化が強すぎて学びにならない
   - 画面下部に小さく表示される「エキスパートモードに切り替える」をクリック
4. 「キャンペーンなしでアカウントを作成」を選択
   - 先にアカウントだけ作り、CVタグ設定後にキャンペーンを作る
5. ビジネス情報を入力
   - 請求先住所の国: 日本
   - タイムゾーン: (GMT+09:00) 東京
   - 通貨: 日本円（JPY）
6. 支払い情報（クレジットカード）を登録

---

## Step 2: コンバージョンアクションの設定

GA4 のキーイベントを Google広告にインポートする。

### 2-1. GA4 と Google広告をリンク

1. Google広告 → 管理者（🔧）→ **リンク済みアカウント**
2. 「Google アナリティクス（GA4）」→ 「詳細」
3. GA4 プロパティ `G-1CF4H5GXSM` を選択 → 「リンク」
4. リンクが完了するまで数分待つ

### 2-2. コンバージョンアクションのインポート

1. Google広告 → 目標（🎯）→ **コンバージョン** → 「概要」
2. 「+ 新しいコンバージョンアクション」
3. 「インポート」→ 「Google アナリティクス 4 プロパティ」→ 「続行」
4. 以下の2つをインポートする:

| イベント名 | 用途 | カウント方法 |
|---|---|---|
| **`generate_lead`** | 見積もり完了（入札最適化の対象） | イベントごとに1回 |
| **`contact_submit`** | お問い合わせ送信（最終KPI） | イベントごとに1回 |

5. **入札最適化の対象（主コンバージョン）の設定:**
   - `generate_lead` → **主アクション**（入札に使用: はい）
   - `contact_submit` → **補助アクション**（入札に使用: いいえ）
   - **理由:** 超ミニマム期は月間CV数が少ないため、`generate_lead` で最適化。`contact_submit` の月間数が30件を超えたら切り替える（§8.4）

---

## Step 3: キャンペーン作成

1. Google広告 → キャンペーン → 「+ 新しいキャンペーン」
2. **キャンペーン目標:** 「見込み顧客の獲得」
3. **キャンペーンタイプ:** 「検索」
4. **コンバージョン目標:** `generate_lead` が選択されていることを確認
5. **キャンペーン名:** `[検索] AI見積もり_超ミニマム`

### 予算と入札戦略

| 項目 | 設定値 |
|---|---|
| **日予算** | 5,000円（月約15万円。慣れたら調整） |
| **入札戦略** | 「コンバージョン数の最大化」 |
| **目標CPA（任意）** | **最初は設定しない**（データ不足で機械学習が回らない。2〜4週間のデータ蓄積後に設定） |

### ネットワーク設定

| 項目 | 設定値 |
|---|---|
| **Google検索ネットワーク** | オン |
| **Google検索パートナー** | **オフ**（品質が低いため） |
| **ディスプレイネットワーク** | **オフ**（検索広告に集中） |

### 地域・言語

| 項目 | 設定値 |
|---|---|
| **地域** | 日本（全国）※ 新潟特化の広告グループは別途作成 |
| **言語** | 日本語 |

### スケジュール

| 項目 | 設定値 |
|---|---|
| **広告のスケジュール** | 初期は「終日」でOK（データが溜まったら平日昼間に絞る等の調整） |
| **開始日** | 準備完了後の日付を指定 |

### 広告グループ 2: 地域特化（新潟）

## Step 5: 広告文（RSA）の作成

各広告グループに1つずつRSA（レスポンシブ検索広告）を作成する。

### 顕在層グループ用

**見出し（最大15本、最低5本入力。各30字以内）:**

| # | 見出し | 備考 |
|---|--------|------|
| 1 | AI見積もり｜最短1分で完了 | 即時性 |
| 2 | システム開発費を無料で概算 | 手軽さ |
| 3 | AI活用でコスト削減を実現 | コスト訴求（審査対応済み） |
| 4 | 中間マージンなしの直接開発 | 価格構造の透明性 |
| 5 | 要件が固まらなくてもOK | 不安解消 |

> ※「相場の1/2」「40%削減」等の具体数値は根拠不足で審査落ちリスクがあるため広告文では使用しない（§12.1）

**説明文（最大4本、最低2本入力。各90字以内）:**

| # | 説明文 |
|---|--------|
| 1 | 選ぶだけで概算費用・内訳・体制・スケジュールまで提示。稟議の根拠にもなるPDFを自動生成。まずは無料で試せます。 |
| 2 | AIが要件を分析し、従来型開発とAIハイブリッド開発の費用を比較。開発計画書まで最短1分で自動生成。そのまま稟議書として活用可能です。 |

**最終URL:** `https://ai.cloudnature.jp/`

### 新潟グループ用

見出しを一部差し替える:

| # | 見出し |
|---|--------|
| 1 | 新潟のシステム開発、1分で概算 |
| 2 | 新潟の中小企業向けAI開発支援 |
| 3〜5 | 顕在層と同じものを流用 |

## Step 6: 広告表示オプション（アセット）の設定

キャンペーン → アセット → 「+」で追加

### サイトリンク（4本以上推奨）

| リンクテキスト | 説明文1 | 説明文2 | URL |
|---|---|---|---|
| よくある質問 | AI見積もりの仕組みや | 料金について解説 | `https://ai.cloudnature.jp/#faq` |
| 会社概要 | 新潟の伴走型IT企業 | クラウドネイチャー | `https://cloudnature.jp/company` |
| 導入事例 | AI導入の実績を | ご紹介します | `https://cloudnature.jp/cases` |
| 無料相談 | まずはお気軽に | ご相談ください | `https://cloudnature.jp/contact` |

### コールアウト（4〜6個）

```
最短1分
PDF出力対応
稟議利用可
無料で概算
```

### 構造化スニペット

- **ヘッダー:** サービスの種類
- **値:** Web開発, モバイル開発, AI開発, 業務自動化

## Step 8: 最終確認・配信開始

### 配信前チェックリスト

- [ ] エキスパートモードになっているか
- [ ] コンバージョンアクションが正しくインポートされているか
- [ ] `generate_lead` が主コンバージョン（入札に使用: はい）になっているか
- [ ] ディスプレイネットワークがオフになっているか
- [ ] 検索パートナーがオフになっているか
- [ ] 日予算が意図通りか（5,000円/日 = 約15万円/月）
- [ ] キーワードがフレーズ一致（"ダブルクォーテーション"）で入力されているか
- [ ] 除外キーワードが設定されているか
- [ ] 広告表示オプション（サイトリンク/コールアウト/構造化スニペット）が設定されているか
- [ ] 最終URLが `https://ai.cloudnature.jp/` になっているか
- [ ] 支払い情報が登録されているか

### 配信開始後（初日〜1週間）

1. **審査通過の確認**（通常24時間以内）
   - 広告のステータスが「有効」になっていることを確認
   - 審査落ちした場合は理由を確認して修正・再審査
2. **GA4リアルタイムレポート** で広告経由のアクセスが来ているか確認
3. **初週は触らない** — データ蓄積期間。CPCが高くても焦らない

### 配信開始後（週次 — #15）

- Google広告 → キャンペーン → 主要指標を確認:
  - **CTR**: 5%以上が目標
  - **CPC**: 500円以下が目標（初期は高めでも許容）
  - **コンバージョン数**: `generate_lead` の件数
- 検索語句レポート → 不要な語句を除外キーワードに追加
- 2〜4週間後: CPA実績が見えてきたら「目標CPA」を入札戦略に設定

---

## 注意事項

- **広告文に「相場の1/2」「40%削減」は使わない** — 根拠不足で審査落ちリスクあり。LP側にも条件付記が必要（#5）
- **「最短1分」は「※選択式の場合の目安」と注記** — 広告文ではなくLPの注記で対応
- **初期2週間はデータ蓄積期間** — CPAが高くても入札戦略や予算を大きく変えない
- **月間CV数が少ない間は目標CPAを設定しない** — 自動入札の学習に最低30件程度のCVが必要
