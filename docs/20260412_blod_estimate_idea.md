# ai.cloudnature.jp SEO戦略書（本体サイトで発信する案）

## 1. 現状分析と課題

### サイト構造の問題

ai.cloudnature.jpは現在、AI見積もりシミュレーターのシングルページ構成になっている。titleタグ（「AI見積もりシミュレーター｜システム開発費を最短1分で自動算出」）はツール名のみで、「新潟」も「中小企業」もタイトルに含まれていない。つまり、地域キーワードでの上位表示がほぼ不可能な状態。

加えて、以下の点が大きな機会損失になっている。

- **コンテンツページが存在しない**：ブログ、事例、解説ページなどが一切なく、検索エンジンが評価するテキスト量が極端に少ない
- **地域シグナルの欠如**：「新潟」「中小企業」などのキーワードがフッターの住所以外にほぼ登場しない
- **内部リンク構造がない**：1ページしかないため、トピッククラスターやサイロ構造を形成できていない
- **構造化データ未実装**（推定）：LocalBusiness、FAQPage、SoftwareApplicationなどのschema.orgマークアップがない

### 競合環境

「新潟 AI導入」「新潟 システム開発」などのキーワードは、以下の3層に支配されている。

**第1層：ポータル・アグリゲーター（非常に強い）**
PRONIアイミツ、システム幹事、比較ビズ、StockSunなどの全国規模マッチングサイト。ドメインパワーが圧倒的で、ヘッドキーワードでは勝ち目がない。

**第2層：地元有力企業（強い）**
にいがたAIビジネス（nabai.jp）、新潟人工知能研究所（NAIL/nai-lab.com）、メビウス（mob.co.jp）、ドコドア（docodoor.co.jp）。コンテンツ量・被リンク・ブランド認知で先行。

**第3層：行政・メディア**
新潟県庁のAI・IoT関連ページ、新潟日報関連記事。権威性が高い。

### 戦略の方向性

ヘッドキーワード（「新潟 システム開発」等）で短期的に勝つのは現実的でない。**ロングテールキーワード × 地域特化コンテンツ**で確実にインデックスを増やし、トピッククラスターを形成していく戦略が正解。

### その他のテクニカル施策

- **サイトマップXMLの設置**：全ページをsitemap.xmlに登録、Google Search Consoleに送信
- **パンくずリスト**：BreadcrumbList schemaとともに全ページに実装
- **OGP / Twitterカードの最適化**：ページごとに固有の画像・descriptionを設定

- **robots.txt**：/complete（結果ページ）と /api/ を Disallow に設定。/chat は「見積もりツール × 無料」の流入入口として index 対象とする

## 11. 競合との差別化ポイント

CloudNatureがコンテンツで打ち出すべき独自の優位性：

1. **AI見積もりシミュレーターという唯一無二のツール**：新潟でこのようなツールを提供している会社は他にない。これ自体がコンテンツの核になる
2. **AI×要件定義の専門性**：コーディングではなく「経営課題を技術に翻訳する力」がE-E-A-Tの根拠
3. **補助金申請のスピード**：Claude活用で3時間で申請書作成という実績は差別化要素
4. **中間マージンゼロの価格構造**：大手SIerとの比較で明確な優位性
5. **新潟特化・対面重視**：東京の会社にはできない「現場に出向いてヒアリング」
