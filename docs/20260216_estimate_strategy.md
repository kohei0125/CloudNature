# 次世代AI見積もり・要件定義プラットフォーム Webサイト設計書

## 1. Webサイト要件定義（既存のmiryo.aiのサイトを分析して、改善案を盛り込んだ設計書）

* **サイトの目的:**
* AIを用いたシステム見積もり・WBS自動生成サービスの価値を伝え、質の高いリード（商談・デモ予約）を獲得する。
* 既存サービス（miryo.ai等）の弱点である「ブラックボックス化」や「信頼性の欠如」を払拭し、エンタープライズ企業にも通用する強固なトラスト（信頼）を構築する。


* **ターゲットユーザー:**
* ITリテラシーが必ずしも高くない企業の事業責任者、経営層。
* 新規事業を立ち上げるスタートアップのCXO。
* 既存のSIerの遅い見積もりスピードや不透明な価格に不満を持つプロジェクトマネージャー。


* **コンバージョン（CV）目標:**
* プライマリ: 対話型AIによる「無料オンライン見積もり（サンドボックス）」の実行と、それに伴うリード情報（メアド・要件）の取得。
* セカンダリ: 専門コンサルタントへの「無料相談・デモ予約」、および「ホワイトペーパー（資料）」のダウンロード。



## 2. サイト構造（サイトマップ）と下層ページの役割

サイト全体の構造は、ユーザーの認知から検討、そしてCVへと自然に誘導するSaaSランディングページのベストプラクティスに従い、ノイズを排除したクリーンな設計とします。

1. **TOPページ:** 価値提案（バリュープロポジション）、機能概要、ソーシャルプルーフ（社会的証明）、インタラクティブな見積もり体験。
2. **How it Works（選ばれる理由/テクノロジー）:** AIハイブリッドモデルや確率論的アプローチなど、ブラックボックスを解消する技術解説。
3. **導入事例（Case Studies）:** ROI（投資対効果）を定量的に証明するページ。
4. **料金モデル（Pricing）:** 固定価格、準委任（T&M）、ハイブリッドなど、柔軟な契約形態の提示。
5. **企業情報（About Us）:** 3C分析に基づき、単なるAIツールではなく専門家集団であることを証明するトラストビルディングの要。
6. **リソース（FAQ / ホワイトペーパー）:** 顧客の不安を先回りして解消し、リードを獲得するコンテンツ。

## 3. 各ページの画面設計と訴求テキスト（コピーライティング）

### ① TOPページ（ランディングページ）

最初の5秒間でユーザーの心を掴み、離脱を防ぐためのヒーローセクション（最上部）を設計します。

* **メインコピー（H1）:**
* *「次世代AIが導く、確実なシステム開発。最短1分で、精緻な見積もりと要件定義を。」*


* **サブコピー:**
* *「過去数万件のプロジェクトデータに基づくハイブリッドAIが、不確実な開発コストを確率論で可視化。SIerの1分の1のコストと圧倒的なスピードで、あなたのアイデアを稟議可能な設計書に変換します。」*


* **CTA（Call to Action）ボタン:**
* コントラストの強い色を使用し、視覚的に際立たせることでコンバージョン率を向上させます。
* テキスト: *「無料でAI見積もりを試す（アカウント登録不要）」*


* **インタラクティブ・プレビュー（サンドボックス）:**
* ヒーローセクションの直下に、ユーザーがスライダーで「アプリの種類」「想定ユーザー数」を動かすと、リアルタイムで「概算コスト（例: 500万円〜800万円）」「開発期間」が動的に変化する簡易シミュレーターを配置します。これにより、プロダクト主導型成長（PLG）のフックとします。


* **ソーシャルプルーフ（社会的証明）:**
* 「導入企業ロゴ」「顧客の評価（平均4.8/5.0など）」を配置し、早期に信頼を獲得します。



### ② 導入事例ページ（Case Studies）

単なるポートフォリオではなく、経済的価値を証明する場とします。

* **ページタイトル:** *「AIがもたらした、劇的なROIとプロジェクトの成功」*
* **訴求方法:**
* **Before/Afterの可視化:** 「従来のSIerが提示した見積もり（期間3ヶ月・1,500万円）」vs「当プラットフォームの提示（期間1ヶ月・500万円）」という数値を明確に比較します。
* **顧客の生の声:** 実際のクライアントの顔写真と詳細なインタビュー（テスト要件の自動化でQAコストがどう下がったか等）を掲載し、信頼性を担保します。



### ③ 企業情報ページ（About Us）

AIシステムに対する「誰が運営しているか分からない」という不安を払拭するための下層ページです。

* **ページタイトル:** *「テクノロジーと専門家の融合。私たちが選ばれる理由」*
* **訴求方法:**
* **専門家チームの顔出し:** データサイエンティスト、機械学習エンジニア、品質保証（QA）エキスパートなど、システムの裏側にいる「人間」の高度な専門性をアピールします。
* **セキュリティとコンプライアンス:** エンタープライズ企業が最も気にするデータ保護方針や、AIによるハルシネーション（もっともらしい嘘）をどのように防いでいるかを明記します。



### ④ リソース＆FAQページ（トラストビルディング）

見積もりの根拠に対する疑問や、開発手法への不安を解消します。

* **ページタイトル:** *「システム開発の不確実性をゼロへ」*
* **訴求方法:**
* **詳細なFAQ:** 「なぜこんなに安いのか？」「途中で要件が変わった場合はどうなるか？」「提示された見積もりの確率幅（70%の確率でこの予算に収まる等）の根拠は？」といった、都合の悪い疑問にも透明性を持って回答します。
* **リード獲得（ホワイトペーパー）:** 「失敗しない要件定義の進め方」「AIを活用したシステム開発コスト削減ガイド」などのPDF資料を用意し、メールアドレスと引き換えに提供することで、まだ検討段階のユーザー（潜在層）のリードを獲得します。



## 4. UI/UXとコンバージョン最適化のベストプラクティス

* **1ページ1ゴール（Single Goal）の原則:** SaaSのランディングページでは、複数の目的を持たせず、「AI見積もりの実行」という単一のゴールにユーザーを集中させます。
* **入力フォームの最適化:** AIとの対話フォームや問い合わせフォームは、入力項目（フィールド）を極限まで減らします。必要な情報だけを段階的に聞くUIにすることで、完了率（CVR）が大幅に向上します。
* **モバイル最適化:** 意思決定者や経営層は移動中にスマートフォンでサイトを閲覧することが多いため、モバイル端末でのレスポンシブデザインと、タップしやすいUIを完璧に構築します。
* **明確なメリット提示（Quick Wins）:** 機能（例：プロンプトエンジニアリングによるWBS生成）をそのまま伝えるのではなく、得られる結果（例：数週間かかる要件定義書が1分で手に入る）という「ベネフィット」に変換して見出しを作成します。

この設計書に基づきWebサイトを構築することで、競合サイトの持つ「手軽さ」という武器を無効化し、「圧倒的な透明性・信頼性・エンタープライズ対応力」を兼ね備えた市場のゲームチェンジャーとなることができます。
