### #3b GA4・Search Console 事前確認（広告開始前に必ず実施）

#### GA4 基本データ取得の確認

1. GA4管理画面 → **プロパティ設定 → データストリーム**
   - 共有プロパティ `G-1CF4H5GXSM` 側で `cloudnature.jp` / `ai.cloudnature.jp` の流入が確認できること
   - 切り分けは `hostName` または `page_location` で行う
2. GA4 → **リアルタイムレポート**
   - cloudnature.jp にアクセス → アクティブユーザー「1」が表示されるか
   - ai.cloudnature.jp にアクセス → 同様に表示されるか
   - **見積もりサイト側で表示されない場合** → 公開中HTMLに `G-1CF4H5GXSM` が埋め込まれているか確認
3. GA4 → **レポート → エンゲージメント → イベント**
   - 過去のイベントデータ（page_view等）が記録されているか
   - データが0の場合、GA4の設置自体に問題がある

#### Search Console の確認

1. **両サイトがプロパティとして登録されているか**
   - `cloudnature.jp` のプロパティ
   - `ai.cloudnature.jp` のプロパティ（サブドメインは別途登録が必要）
   - 未登録の場合は追加する
2. **インデックス状況の確認**
   - Search Console → ページ → インデックス登録済みページ数
   - コーポレートサイト: 8ページ + 記事ページ + ユースケースページ
   - 見積もりサイト: `/` `/chat` `/complete` 程度
3. **広告LP（ai.cloudnature.jp）のインデックス確認**
   - URL検査ツールに `https://ai.cloudnature.jp/` を入力
   - 「URLはGoogleに登録されています」と表示されるか
   - 未登録の場合「インデックス登録をリクエスト」を実行
   - **Google広告の審査でLPがインデックス済みであることが前提になる場合がある**
4. **robots.txt の確認**
   - `https://ai.cloudnature.jp/robots.txt` にブラウザでアクセス
   - 広告LP（`/`）がDisallowされていないこと
   - `https://cloudnature.jp/robots.txt` も同様に確認

### #3a GA4イベントの確認手順

1. Chrome DevTools の Console で `npm run dev` のローカル環境を開く
2. Console に `window.gtag` と入力し、関数が存在することを確認
3. 見積もりフローを1回通す。各ステップで Console に以下を入力して直前のイベントを確認:
   - `dataLayer.filter(e => e[0] === 'event')` でイベント一覧を確認
4. **GA4 DebugView で確認**（本番デプロイ後）:
   - GA4管理画面 → 管理 → DebugView
   - Chrome拡張「[Google Analytics Debugger](https://chrome.google.com/webstore/detail/google-analytics-debugger/jnkmfdileelhofjcijamephohjechhna)」をON
   - ai.cloudnature.jp で見積もりフローを通す
   - DebugView に estimate_start → estimate_step(×12) → generate_lead → view_estimate_complete が出ること
   - cloudnature.jp/contact でフォーム送信 → contact_submit が出ること
5. **GA4 リアルタイムレポート** でもイベントが表示されることを確認

### #6a GTM動作確認手順

> **重要:** 現行運用では `cloudnature.jp` と `ai.cloudnature.jp` を共有GA4 `G-1CF4H5GXSM` に統一する。
> GTM切替時も、先にこの測定IDでGA4設定タグを作成してから公開すること。

1. GTMコンテナを作成し、コンテナID（GTM-XXXXXXX）を取得
2. **GTM管理画面で先にGA4設定タグ（G-1CF4H5GXSM）を作成・公開する**（これを忘れるとGA4データが途絶える）
3. Vercelの環境変数に `NEXT_PUBLIC_GTM_ID=GTM-XXXXXXX` を設定（両サイト）
4. **まず1サイト（estimateを推奨）だけデプロイし、GA4リアルタイムレポートでデータが来ることを確認**
5. 問題なければもう1サイトもデプロイ
6. [GTM Tag Assistant](https://tagassistant.google.com/) で両サイトのGTMコンテナ読み込みを確認
7. 旧gtag.js直接読み込みは `NEXT_PUBLIC_GTM_ID` 設定時に自動で無効化されるので二重計測にならない

**ロールバック手順:** GTM切替後に問題があれば、Vercelから `NEXT_PUBLIC_GTM_ID` を削除して再デプロイ。元のgtag.js直接読み込みに自動で戻る
