# AI活用の壁打ち受付フォーム仕様 v2（cloudnature.jp用）

対象：Threadsプロフィールリンク①「30分だけAI活用の壁打ち（無料）」の着地先
v1からの変更：①「営業しません」→「無理な提案はしません」（整合性修正）②datetime入力を廃止し時間帯チェックボックスに ③実施方法の項目を削除（リード文で処理）④必須を本当に3項目に ⑤「予約」→「受付」⑥リードDB強化（投稿DBリレーション追加）

## 1. ページ文言（コピペ用）

**ページタイトル**
```
30分だけ、AI活用の壁打ち（無料）
```

**リード文**
```
「AIって、結局うちの会社で使えるの？」
その段階で大丈夫です。

まだ何も決まっていなくても、漠然としていても構いません。
30分だけ、普段の業務の話を聞かせてください。

基本はオンライン（Google Meet）で実施します。
新潟市内の場合は、内容に応じて対面もご相談ください。

無理な提案はしません。
話してみて、AIよりExcelや既存ツールで十分そうであれば、そのままお伝えします。
```

**送信ボタン**
```
壁打ちを申し込む
```

**プライバシー文言（送信ボタン直上）**
```
ご入力いただいた情報は、壁打ちのご連絡と実施準備のみに利用します。
```

## 2. フォーム項目定義（必須3項目）

| # | 項目名 | 形式 | 必須 | 選択肢・placeholder | 目的 |
|---|---|---|---|---|---|
| 1 | お名前 | text | ✅ | placeholder：山田 太郎 | 連絡 |
| 2 | メールアドレス | email | ✅ | placeholder：you@example.com | 連絡 |
| 3 | 話したいこと（複数選択可） | checkbox | ✅ | 下記A参照 | サービス分岐・事前準備 |
| 4 | 会社名・屋号 | text | 任意 | placeholder：例）株式会社◯◯、◯◯商店（個人の方は空欄でOK） | 資格確認 |
| 5 | お立場 | select | 任意 | 経営者・役員／個人事業主・フリーランス／会社員（決裁に関わる）／会社員（現場・担当）／その他 | 資格確認・案件化判断 |
| 6 | 従業員数 | select | 任意 | 1人／2〜10人／11〜50人／51〜100人／101人以上 | ターゲット適合 |
| 7 | いま一番時間がかかっている業務 | textarea（2行） | 任意 | placeholder：例）見積もり作成、月30件で1件2時間くらい | 棚卸しの種。壁打ち当日の準備に直結 |
| 8 | 都合のつきやすい時間帯（複数選択可） | checkbox | 任意 | 平日午前／平日午後／平日夕方／土日／いつでも調整可 | 日程調整の下準備 |
| 9 | 知ったきっかけ | select | 任意 | Threads／ご紹介／検索／セミナー・イベント／その他 | UTM欠損時のバックアップ |

※日時の確定はフォームで取らず、送信後にメールで調整（初期は手動。月10件超でGoogleカレンダー予約スケジュール統合を検討）。
※実施方法（オンライン/対面）の項目は置かない。リード文で「基本オンライン・対面は相談」と伝え、対面希望は日程調整メールで拾う。

**A. 「話したいこと」の選択肢（短縮版）**
```
□ 特定の業務を楽にしたい
□ 社内にAIを定着させたい
□ システム・AIエージェント開発の相談
□ 社員にAIを学ばせたい
□ まだ漠然としている（それでOKです）
```
※補足（見積もり・請求・日報など、の例示）はフォーム下に小さく表示。選択肢はサービス分岐ロジック（開発／伴走支援／アカデミー）に対応。

## 3. hiddenフィールド（計測用）

| フィールド | 取得方法 |
|---|---|
| utm_source / utm_medium / utm_campaign / utm_content | URLクエリから取得しhidden inputに保持（sessionStorageで維持） |
| referrer | document.referrer |
| 送信日時 | サーバー側で付与 |
| honeypot | スパム対策の不可視項目（入力があれば破棄） |

## 5. リードDB設計（Notion・新設）

投稿DBとは別に「壁打ちリードDB」を新設。

| プロパティ | 形式 | 内容 |
|---|---|---|
| 氏名 | TITLE | ─ |
| メール | EMAIL | ─ |
| 会社名 | TEXT | ─ |
| 話したいこと | MULTI_SELECT | フォーム選択肢と同一 |
| お立場／従業員数 | SELECT | フォーム選択肢と同一 |
| 時間がかかっている業務 | TEXT | 棚卸しの種 |
| 流入元 | TEXT | utm_content＋referrer |
| **初回接点投稿** | **RELATION（→Threads投稿DB）** | どの投稿が案件を生んだかをDB間で接続。分析の要 |
| 紹介者 | TEXT | 地方案件で重要 |
| ステータス | SELECT | 未確定→確定→実施済み→アセスメント提案→案件化／見送り |
| 温度感 | SELECT | A／B／C |
| 想定サービス | SELECT | 開発／伴走支援／研修／アセスメント |
| 次アクション日 | DATE | 追客漏れ防止 |
| メモ | TEXT | 壁打ち後の所感 |

## 6. 実装メモ

- スタック：Next.js（App Router）Route Handler → Resend（自動返信＋内部通知）→ Notion API（リードDB記録）
- バリデーション：メール形式／honeypot
- ページURL案：`cloudnature.jp/kabeuchi`（Threadsリンク①のUTM：`?utm_source=threads&utm_medium=social&utm_campaign=profile&utm_content=kabeuchi`）
- **壁打ちの出口は有償アセスメント提案に固定**（フォームには一切書かない）
- 日程確定メールに対面希望の確認を含める（新潟市内の場合のみ）

## 7. 置かない項目（意図的な除外）

| 除外項目 | 理由 |
|---|---|
| 電話番号 | 心理ハードルが上がる。必要ならメールで確認 |
| 予算・年商 | 壁打ち段階で聞くと営業臭。従業員数で代替 |
| 相談内容の詳細（長文必須） | 書く負担で離脱。「時間がかかっている業務」任意2行で十分 |
| 実施方法（オンライン/対面） | 項目にすると重い。リード文＋日程調整メールで処理 |
| 希望日時（datetime） | スマホで離脱要因。時間帯チェックボックス＋メール調整に変更 |
| 住所 | 不要 |
