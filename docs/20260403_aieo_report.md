## AIEOって何なのか

### SEO vs AIEO（仕組みが根本的に違う）

SEOは「Googleの検索アルゴリズムに最適化する」技術です。10年以上これが主流でした。

AIEOは「ChatGPT、Perplexity、Gemini、ClaudeなどのAIに引用・推薦される」ための最適化です。

- **SEO**
    - クローラーがサイトを巡回
    - インデックスに登録
    - ランキングで表示
- **AIEO**
    - RAGパイプライン（検索→取得→生成）でAIが回答を生成
    - その回答内で引用される

ここで重要なデータがあります。

AEO最適化したコンテンツは、公開から **1〜2週間** で最初のAI引用を獲得できます。

2〜3週間で全体的なメンション率も改善されます。

しかもLLM引用の44.2%は **テキストの最初の30%**（つまりイントロ）から取られてます。導入文の書き方だけで引用されるかどうかが決まるんですよ。

---

### SEOとAIEOの具体的な違い

SEOでは「キーワード密度」「被リンク数」「ドメインパワー」が重要でした。

AIEOでは全く違う要素が効いてきます。

- **信頼シグナル**
    - AIは「他の権威あるソースでも同じ情報が裏付けられているか」を重視します。
    - 1つのサイトで書くだけじゃなく、複数の信頼できるソースから引用される状態を作る必要があります。
- **構造化データ**
    - [Schema.org](http://Schema.org) マークアップを入れることで、AIがコンテンツの意味を正確に理解できます。
    - FAQPage、HowTo、Articleなどのスキーマが特に効果的です。
- **回答性の高い文章**
    - AIは「質問に対して直接答えている文章」を優先的に引用します。
- **データの具体性**
    - 「多くの企業が導入」より「87%の企業が導入（2026年調査）」のほうがAIに引用されやすいです。

---

### 実際にAIに引用されるコンテンツの型

自分が試して効果があったのは、この構造です。

1. **見出しを質問形式**にする
    - 例: 「AI研修の費用相場は？」
2. **最初の1〜2文で端的に回答**する
    - 例: 「法人向けAI研修の費用相場は、1日研修で30〜80万円、3日間パッケージで100〜200万円が一般的です。」
3. **根拠や具体例を箇条書き**で補足する

この構造にすると、Perplexityで「AI研修 費用」と検索されたときに、回答の中で引用されるようになります。

## AIEOが広告運用に影響する理由

### AIが「おすすめ」する時代

今までは「Google検索で1位を取る」か「広告で上位表示する」の2択でした。

でも今は3つ目の選択肢が出てきてます。

**AIが直接「〇〇がおすすめです」と回答する**んですよ。

試しにPerplexityで「法人向けAI研修 おすすめ」と検索してみてください。Google検索とは全然違う形で、AIが「この会社のサービスがいい理由は〇〇」みたいに直接回答してきます。

---

### 広告データがAIEO対策の宝庫

ここが一番面白いところなんですけど、**Google Adsの検索語句レポートはAIEOのネタ元として最強**なんですよね。

自分の場合、こういう発見がありました。

- 「AI研修 比較」で検索してる人が多い
    - 比較記事を書いたらPerplexityで引用された
- 「Claude Code 使い方 初心者」がCTR高い
    - 初心者ガイドを書いたらChatGPTで引用された
- 「生成AI コンサル 費用」のCPCが高い
    - 費用相場記事を書いたら広告費を削減できた

つまり、**広告で集客するだけじゃなく、広告データを使ってAIに推薦される状態を作る**ことで、結果的に広告費が下がるんですよね。

## Claude Codeで広告を自動化する仕組み

### Claude Code → Google Ads / Meta Ads / Ubersuggest の連携フロー

自分が使ってる構成はこうなってます。

- **Google Ads**
    - Python + Google Ads API（GAQL）で直接操作
- **Meta Ads**
    - Graph API v22.0でFacebookとInstagram広告を操作
- **キーワードリサーチ**
    - Ubersuggest Internal API（無料、登録不要）
- **全部Claude Codeのカスタムエージェント**から操作

---

### カスタムエージェントの仕組み

Claude Codeには **--agent** オプションがあって、用途別のカスタムエージェントを定義できます。

具体的には **.claude/agents/[google-ads.md](http://google-ads.md)** というファイルに、こういう内容を書いてます。

- Google Ads APIの接続情報（アカウントID、認証設定のパス）
- GAQL（Google Ads Query Language）のリファレンス
- キャンペーン作成、広告グループ作成、キーワード追加などの具体的な手順
- 安全ルール（「必ずPAUSED状態で作成すること」など）

これを書いておくと、**claude --agent=google-ads** で起動したときに、エージェントがこの知識を全部持った状態で動いてくれます。

---

### なぜMCPじゃなくPython直接実行なのか

広告APIは操作が複雑で、MCP（Model Context Protocol）経由だと柔軟性が足りないケースが多いんですよね。

Google Ads APIはGAQLという独自のクエリ言語があって、SQLっぽく書けます。これをPythonスクリプトとして直接実行するほうが、エラーハンドリングも含めて確実に動きます。

---

## 実際にどうやっているか

### キーワード調査（Ubersuggest API）

まずUbersuggest APIで、狙いたいキーワードの月間検索ボリューム、CPC、SEO難易度を一括取得します。

Ubersuggest APIの面白いところは、**無料で、登録も不要**なんですよね。Neil Patelのサイトの内部APIをそのまま叩いてるだけなんですけど、Google Keyword Plannerと同じデータが取れます。

自分の場合、キーワードを20〜30個投げて、以下の優先順位で並べ替えてます。

- ボリューム1,000以上 + CPC 300円以上
    - 広告キャンペーン候補
- ボリューム500以上 + SEO難易度40以下
    - AIEO記事の候補
- ボリュームが右肩上がり（3ヶ月で+20%以上）
    - 優先対応

30キーワードの調査が30秒で終わります。

---

### Google Adsキャンペーン作成

キーワードが決まったら、Claude Codeに「このキーワード群で検索キャンペーンを作って」と指示します。

エージェントが自動で以下を行います。

- 日予算を設定（月予算 ÷ 30.4で日割り計算）
- 広告グループを作成（キーワードのテーマごとにグルーピング）
- キーワードを追加（フレーズ一致または完全一致を自動判断）
- レスポンシブ検索広告（RSA）を生成（見出し5個 × 説明文3個）
- **必ずPAUSED状態で作成**（勝手に配信が始まらないように）

正直、この「PAUSED状態で作成して確認を待つ」というルールが一番大事です。

AIに広告運用を任せるとき、**勝手にお金が使われない仕組み**を必ず入れるべきです。

---

### パフォーマンス分析（GAQL）

自分がよく使うクエリです。

- **日次パフォーマンス**
    - 過去7日間のimp、クリック、コスト、コンバージョンを日付別に取得
- **検索語句レポート**
    - 実際にどんな検索キーワードで広告が表示されたかを確認
    - 無駄なキーワードを発見して除外
- **コンバージョン分析**
    - どのキーワード、どの広告がコンバージョンに繋がってるかを特定
- **インプレッションシェア**
    - 自社広告が表示される割合
    - 予算不足で機会損失してないかチェック

特に **検索語句レポート** はAIEO対策にも直結します。

---

### Meta広告（Facebook/Instagram）

Meta Ads側も同じ要領で、Graph API v22.0を使ってClaude Codeから操作してます。

Google Adsとの大きな違いは、Meta広告は **ターゲティングの細かさ** です。

- 年齢、性別、地域
- 「AIに興味がある」「経営者」「IT業界」などの興味関心ターゲティング
- カスタムオーディエンス（Webサイト訪問者のリターゲティング）
- 類似オーディエンス（既存顧客に似た人を自動で探す）

特にInstagramのリール広告は、最近CPAがかなり下がってて費用対効果が良いです。

## AIEOを意識した広告運用の実践ポイント

### 1. LPにFAQ構造を入れる

ランディングページの最下部に、ユーザーがよく聞く質問をFAQ形式で入れてます。

重要なのは [**Schema.org](http://Schema.org)のFAQPageマークアップを同時に実装する** ことです。

自分の場合、「AI研修の費用は？」「期間はどのくらい？」「オンラインでも受けられる？」みたいなFAQをLPに入れたら、Perplexityで「AI研修 費用」と検索されたときに引用されるようになりました。

### 2. 比較記事を公開する

「〇〇 vs △△」系の比較記事は、AIに引用されやすいコンテンツの代表格です。

### 3. 広告データをコンテンツ改善に活かす（循環フロー）

広告データ → AIEO対策 → AI引用 → 広告費削減の循環。

これが一番のポイントです。

- Google Adsの検索語句レポート
- ユーザーの本当の質問を発見
- FAQ付き記事を作成
- PerplexityとChatGPTで引用される
- 広告費なしで集客できる
- 浮いた予算を新しいキーワードの広告に投入

この循環が回り始めると、広告費を減らしながら集客が増えます。

自分の場合、この循環を3ヶ月回した結果、月間の問い合わせ数はほぼ同じなのに広告費が約30%下がりました。

## 自分のAIEO対策の具体的な結果

正直に数字を出すと、AIEO対策を始めてから3ヶ月で以下の変化がありました。

- Perplexityで「AI研修」系キーワードの引用回数
    - 0回 → 月12回
- ChatGPTで自社名が回答に含まれるケース
    - 月3〜4回確認
- オーガニック問い合わせ（広告経由じゃない流入）
    - 月2件 → 月7件
- Google Ads月額費用
    - 約30%削減（AIEOからの流入が増えた分）

特に効いたのが「AI研修 費用相場」の記事です。

FAQ構造 + [Schema.org](http://Schema.org)マークアップで書いたら、公開5日目にPerplexityの回答に引用されてました。「1〜2週間で初回引用」というデータが本当だったんですよね。

逆にうまくいかなかったのは、技術的な内容が深すぎる記事です。

「Claude CodeのHooks設定方法」みたいな記事は、AIに引用はされるんですけど問い合わせには繋がらない。

AIEOは「引用される」だけじゃなく、「引用された結果、ビジネスに繋がる」ところまで設計しないと意味がないです。

## これから始める人へ

AIEOを始めるのに、自分みたいにGoogle Ads APIやMeta Ads APIを全部セットアップする必要はないです。

まず最初にやるべきは3つだけです。

- **既存のWebサイトにFAQ構造を追加する**
    - よくある質問を5〜10個
    - 質問形式の見出し + 端的な回答 + 補足の構造
    - [Schema.org](http://Schema.org)のJSON-LDマークアップも忘れずに
- **Perplexityで自社名を検索する**
    - 今AIがどう回答しているかを確認するだけで、対策の方向性が見えます
- **比較記事を1本書く**
    - 「自社サービス vs 競合A vs 競合B」の比較記事は、AIに引用されやすいコンテンツの定番です

この3つをやるだけで、早ければ1〜2週間でAIの回答に変化が出ます。

広告の自動化はその次でいいと思ってます。

Claude Codeのカスタムエージェント（--agent）は確かに便利なんですけど、まずはAIEOの基盤を作るほうが先です。

基盤ができれば、広告費は自然と下がっていくので。

ただ一度作ってしまえば、あとはエージェントが回してくれます。

キーワード調査もキャンペーン作成もレポート分析も、全部ターミナルから声で指示するだけです。

AIEOに関しては、今始めれば先行者優位が取れると思ってます。
