## 2. 目的

### なぜブログ記事を書くのか

1. **ToB向け検索流入の獲得**: 「AI導入支援」「AI 業務効率化 中小企業」等のキーワードで検索されたとき、記事がCloudNatureへの入口になる
2. **E-E-A-T（経験・専門性・権威性・信頼性）の蓄積**: 自社の実践ノウハウを記事化し、Googleおよび AI検索（AI Overviews等）から評価される状態を作る
3. **営業資産の転用**: 営業トークスクリプトで使っている12の「AIが使えるパターン」をコンテンツ化し、オンラインでも同じ説得力を発揮させる
4. **無料診断（CTA）への導線**: 各記事から自然に「御社の場合はどうか、無料診断で確認」へつなげる

### ゴール指標

- 公開6ヶ月後: 12記事合計で月間オーガニック流入 500 PV以上
- 記事経由の無料診断申し込み: 月2〜3件

## 3. 営業トークスクリプトの原則（記事にも貫く）

CloudNatureの営業トークスクリプトから抽出した原則です。
**すべての記事でこの原則を守ること**。

### 3つの鉄則

1. **「何ができる？」に直接答えない → まず課題を聞く**: 記事でもリード文は「こんなお悩みありませんか？」から入る。機能説明から始めない
2. **「その場でデモ」が最大の武器 → 記事ではプロンプト例や具体手順がそれにあたる**: 読者が「自分でもできそう」と感じる具体性を提供する
3. **「できない」で終わらない → 必ず代替案・次のアクションにつなぐ**: AIの限界も正直に書いたうえで、「それはシステム開発で対応できます」「まず無料診断で確認しましょう」とつなげる

### 絶対NGワード・表現

- 「なんでもできます」→ 信頼を失う
- 「DXプロジェクト」→ 大きすぎて中小企業に刺さらない
- 「AIの機能リスト」→ 読者の頭に残らない。必ず「御社の場合は〜」に変換
- 一般論のプレゼンで終わること → 必ず具体的な手順・数字・事例を入れる

### 使うべきフレーズ

- 「御社の場合は〜」（一般論を自分ごとにする）
- 「まず1業務だけ」（スモールスタート）
- 「2週間で動くもの」（具体的な期間）
- 「当たり前になっている作業ほど、削減余地が大きい」

## 4. 記事の共通フレームワーク

すべての記事に適用する構成パターンです。

### 構成テンプレート

```
H1: [記事タイトル]

リード文（200〜300字）
- 課題への共感（「こんなお悩みありませんか？」）
- この記事で分かることの要約
- 対象読者の明示

H2: [課題の深掘り・数字で「もったいなさ」を可視化]
- 「週◯時間 = 月◯人月分」の換算
- 営業スクリプトSTEP3の手法をそのまま使う

H2: [AIでできること・できないことを正直に整理]
- できること: 具体的な機能・効果
- できないこと: AIの限界（必ず書く）
- 「AIは下書き・分類まで、最終確定は人」という設計思想

H2: [実践ステップ（手順・プロンプト例）]
- 明日から試せる具体的な手順
- コピペで使えるプロンプト例（最低1つ）
- 使用ツール名を明記（ChatGPT / Claude / Dify / n8n 等）

H2: [応用・自動化のヒント]（任意）
- n8nやDifyを使った自動化パイプラインの紹介
- 「手作業→半自動→全自動」の3段階

H2: [まとめ + CTA]
- 記事の要点を3行で
- CTA: 「御社の業務でどこまで自動化できるか、無料診断でお伝えします」
- 内部リンク: 関連サービスページ + 他の関連ブログ記事
```

### 文体・トーン

- **です・ます調**（丁寧だがフランクに寄せる）
- 専門用語は初出時に必ず1行で説明する
- 1文は60字以内を目安。長文を避ける
- 太字は見出しとキーワードのみ。乱用しない
- 「！」は記事全体で2回まで
- 絵文字は使わない

### SEO要件

- タイトルは32〜40字。狙うKWを前方に配置
- H2見出しにもKWを自然に含める
- meta description用の要約文（120字以内）を記事末尾に付記する
- 画像のalt textの提案を各記事に2〜3個含める

### 文字数

- 1記事あたり 3,000〜5,000字（H1・見出し除く本文のみ）

### 内部リンク

各記事に以下を必ず含める:
- 関連するサービスページへのリンク（1〜2本）
- 関連する他のブログ記事へのリンク（公開済みのものから1〜2本）
- CTAとして /contact または https://ai.cloudnature.jp/chat へのリンク

H2: まとめ｜「まず1業務だけ」で十分です

## 7. 品質チェックリスト

記事完成後、以下をすべて確認すること。

### 内容

- [ ] リード文が「課題の共感」から始まっている（機能説明から始まっていない）
- [ ] 数字で「もったいなさ」を可視化している（「週◯時間 = 月◯人月分」）
- [ ] 「できること」と「できないこと」の両方を書いている
- [ ] 具体的なプロンプト例 or 手順が含まれている

- [ ] 営業スクリプトのNGワードを使っていない
- [ ] 「御社の場合は〜」的な自分ごと化のフレーズがある
